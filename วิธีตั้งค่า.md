# 📋 วิธีตั้งค่าเชื่อม Google Sheets

## ขั้นตอนที่ 1: ตั้งค่า Google Apps Script

1. **เปิด Google Sheets** ของคุณ:
   👉 https://docs.google.com/spreadsheets/d/1HYA_HvAL2XLs521gEV_jcRyE3e3IvkO5lveACZwZfoQ/edit

2. **ไปที่เมนู** `Extensions` (ส่วนขยาย) → `Apps Script`

3. ในหน้า Apps Script Editor:
   - **ลบโค้ดเดิม** ทั้งหมดใน `Code.gs`
   - **คัดลอกโค้ดจากไฟล์** `apps-script.gs` แล้ววางลงไป
   - **กด Save** (Ctrl+S)

4. **Deploy เป็น Web App**:
   - คลิก `Deploy` → `New deployment`
   - คลิกไอคอนเฟืองข้าง "Select type" → เลือก `Web app`
   - ตั้งค่า:
     - **Description**: ระบบรับเรื่องร้องเรียน
     - **Execute as**: Me (ตัวของฉัน)
     - **Who has access**: Anyone (ทุกคน)
   - กด **Deploy**
   - ระบบจะขอสิทธิ์ → กด **Authorize access** → เลือกบัญชี Google → กด **Allow**

5. **คัดลอก URL** ที่ได้ (Web app URL)
   - จะเป็นรูปแบบ: `https://script.google.com/macros/s/AKfycbx.../exec`

## ขั้นตอนที่ 2: ตั้งค่า URL ในเว็บ

1. **เปิดไฟล์** `script.js`

2. **แก้ไขบรรทัดที่ 3** จาก:
   ```javascript
   const APPS_SCRIPT_URL = 'YOUR_APPS_SCRIPT_URL_HERE';
   ```
   เป็น:
   ```javascript
   const APPS_SCRIPT_URL = 'https://script.google.com/macros/s/AKfycbx.../exec';
   ```
   (ใส่ URL ที่คัดลอกมาจากขั้นตอนที่ 1)

3. **Save** ไฟล์

## ขั้นตอนที่ 3: ทดสอบ

1. เปิด `index.html` ในเบราว์เซอร์
2. กรอกฟอร์มทดสอบ แล้วกดส่ง
3. เช็คที่ Google Sheets ว่ามีข้อมูลเข้ามาหรือไม่
4. ทดสอบตรวจสอบสถานะด้วยเลข REF ที่หน้าแรก

## 💡 หมายเหตุ

- **ข้อมูลจะเก็บ 2 ที่**: localStorage (สำรอง) + Google Sheets (หลัก)
- **ตรวจสอบสถานะ**: ใส่เลข REF ที่หน้าแรกของเว็บ
- **จัดการเรื่องร้องเรียน**: เปิด Google Sheets โดยตรง แก้ไขคอลัมน์ "สถานะ" และ "หมายเหตุแอดมิน"
- **สถานะที่ใช้ได้**: `รอดำเนินการ`, `กำลังดำเนินการ`, `เสร็จสิ้น`

## 🔧 ถ้ามีปัญหา

### Deploy ไม่ได้
- ตรวจสอบว่าล็อกอินบัญชี Google ที่เป็นเจ้าของ Sheet
- ลองกด Deploy → Manage deployments → ลบแล้ว Deploy ใหม่

### ข้อมูลไม่เข้า Sheet
- เปิด Developer Console ของเบราว์เซอร์ (F12) → ดูข้อความ error ที่ Console
- ตรวจสอบว่า URL ถูกต้อง
- ลอง Deploy ใหม่ด้วย "New deployment" (ไม่ใช่ Edit existing)

### ตรวจสอบสถานะไม่เจอ
- ถ้าใช้ `no-cors` mode ระบบจะตรวจสอบจาก localStorage ก่อน
- ข้อมูลที่ส่งผ่าน Google Sheets จะตรวจสอบได้เมื่อตั้งค่า URL ถูกต้อง
